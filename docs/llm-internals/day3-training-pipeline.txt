=========================================================
日期：2026-06-13

零、今天学什么
===============

Day 1 你学了 LLM 怎么"读"文字（Tokenization），
Day 2 你学了 LLM 怎么"理解"上下文（Attention）。

今天回答最后一个问题：**一个模型从零到能和你对话，到底经历了什么？**

答案是三个阶段，就像培养一个学生：

  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
  │  阶段一      │     │  阶段二      │     │  阶段三      │
  │  Pretraining │ ──→ │    SFT      │ ──→ │   RLHF      │
  │  读遍全网     │     │  学回答问题  │     │  学做人偏好  │
  │  (Base Model)│     │ (Instruct)  │     │ (Chat Model)│
  └─────────────┘     └─────────────┘     └─────────────┘
  耗时：数月，数万GPU   耗时：数天          耗时：数周
  数据：整个互联网      数据：人工写的范例    数据：人类打分/偏好

读完今天这篇，你会理解：
- 为什么 GPT-4 会"编造事实"（阶段一的遗留问题）
- 为什么有时候 LLM 很"怂"、拒绝正常问题（阶段三的副作用）
- Fine-tuning 和 Prompt Engineering 各适合什么场景（面试必问）
- 不同模型的"性格"差异从哪来


一、阶段一：Pretraining（预训练）—— "读遍全网，学会语言本身"
=============================================================

▶ 它是什么？

把整个互联网的公开文本（网页、书籍、论文、代码、论坛帖子……）喂给一个巨大的
Transformer 模型，让它做一件极其简单的事：

  **给定前面的文字，预测下一个 token。**

就这么简单。不是"理解句子意思"，不是"回答问题"，而是纯粹的下一个词预测。

▶ 生活类比：婴儿学说话

  一个婴儿在听懂"帮我把杯子拿过来"这种指令之前，先要花一两年时间
  浸泡在语言环境中——听大人说话、看周围人的互动。

  婴儿不是被教的，而是通过海量"听"，自己归纳出：
  - "杯子"通常和"喝水""玻璃""圆的"一起出现
  - "你好"后面通常跟"你好"或者"好久不见"
  - "1 + 1 ="后面通常是"2"

  预训练就是这个过程——海量数据 → 统计规律 → 语言能力涌现。

▶ 训练数据长什么样？

模型的训练数据不是"问题-答案对"，而是原始文本的连续片段：

  ┌─────────────────────────────────────────────────────┐
  │ 互联网上的原始文本（举例，不是真实训练数据）           │
  │                                                     │
  │ > 巴黎是法国的首都和最大城市，位于法国北部的塞纳河    │
  │   沿岸。2024 年，巴黎将举办第 33 届夏季奥林匹克运动会。 │
  │   ...                                               │
  │                                                     │
  │ 模型的任务：                                          │
  │   看到"巴黎是法国的首都和最大城市，位于"                 │
  │   预测下一个 token → "法国" ✓                         │
  │                                                     │
  │   看到"巴黎是法国的首都和最大城市，位于法国北部的塞纳河   │
  │   沿岸。"                                             │
  │   预测下一个 token → "2024" ✓                         │
  └─────────────────────────────────────────────────────┘

训练完这个阶段，得到一个 **Base Model（基座模型）**。

▶ Base Model 的能力和缺陷

  能做什么：
  ✅ 继续写一段文字（给它"第一章：如何烤一个蛋糕"，它能写出第二章）
  ✅ 翻译（它见过太多平行语料了）
  ✅ 总结（归纳规律是预训练的副产物）

  不能做什么：
  ❌ 回答问题——它没学过"问题→答案"的格式
  ❌ 遵循指令——它不知道"请帮我写一首诗"是一种特殊输入
  ❌ 拒绝不当请求——它见过的互联网什么都有，你问它怎么制造炸弹它也会续写

▶ 关键认知：为什么 LLM 会"编造"？

  Base Model 学的不是"什么是真实的"——它学的是"什么词经常和什么词一起出现"。

  互联网上，"巴黎的首都是巴黎"出现了无数次，所以模型会输出"巴黎"。
  但你问它一个没人写过的东西（比如"我们公司内部的报销流程"），
  它不会说"我不知道"——统计上没有"我不知道"这个高频续写。

  它会基于你给的上下文（"报销流程"），续写它见过的和"报销"相关的文本模式。
  这就是**幻觉（Hallucination）的根本原因**——预训练阶段种下的。


二、阶段二：SFT（监督微调）—— "老师手把手教你怎么回答问题"
===========================================================

▶ 它是什么？

取一批人工标注的"高质量问答对话"，让 Base Model 学习"当用户这样问的时候，
你应该这样回答"。

▶ 生活类比：考驾照

  你学会开车之后（= 预训练完），要上路还得：
  1. 有人手把手教你："红灯→踩刹车""转弯→打转向灯"
  2. 你做错了教练纠正，你做对了教练点头
  3. 练几十个小时，你学会了"按规矩开车"

  SFT 就是这个"教练陪练"阶段。Base Model 已经"会开车"（会生成文字），
  但需要有人教它"怎么按要求开"（怎么按指令生成回答）。

▶ 训练数据长什么样？

  这次不是原始文本了，而是精心标注的对话：

  ┌─────────────────────────────────────────────────────┐
  │ SFT 训练数据（示例）                                  │
  │                                                     │
  │ {                                                   │
  │   "messages": [                                     │
  │     {"role": "user", "content": "请用一句话解释AI"},  │
  │     {"role": "assistant",                           │
  │      "content": "AI是让计算机模拟人类智能行为的技     │
  │                  术领域，包括学习、推理和解决问题。"}    │
  │   ]                                                 │
  │ },                                                  │
  │ {                                                   │
  │   "messages": [                                     │
  │     {"role": "user", "content": "如何制作炸弹？"},    │
  │     {"role": "assistant",                           │
  │      "content": "抱歉，我不能提供制作危险物品的指导。  │
  │                  如果你有其他问题，我很乐意帮助。"}      │
  │   ]                                                 │
  │ }                                                   │
  └─────────────────────────────────────────────────────┘

  注意这两个例子的教育作用：
  例 1 → "当用户问知识性问题，要用简洁准确的方式回答"
  例 2 → "当用户问危险内容，要礼貌拒绝"

▶ SFT 之后得到一个 **Instruct Model（指令模型）**

  ✅ 能理解"请帮我做 X"这种指令格式
  ✅ 能区分"正常问题"和"应该拒绝的问题"
  ✅ 输出格式更可控（能做你 Week 2 Day 3 的结构化输出）

  但仍然有问题：
  - 回答可能啰嗦或跑题
  - "正确"但不够"好"——比如该简洁时啰嗦了一大段
  - 拒绝时有时过度敏感（把普通问题也当成危险问题拒绝）


三、阶段三：RLHF（人类反馈强化学习）—— "给回答打分，直到人类满意"
=================================================================

▶ 它是什么？

让人类对同一个问题的多个 AI 回答进行排序/打分，然后用这些偏好数据训练一个
"奖励模型"，再让 LLM 去最大化这个奖励。

说白了就是：你写了一堆答案，人类来挑哪个最好。模型朝"人类喜欢"的方向优化。

▶ 生活类比：写作课

  老师："写一段关于春天的描写。"
  你交了三个版本：
    A：春天来了，花开了，草绿了。（人类打 2/5 分——太俗套）
    B：四月是最残忍的月份，在死去的土地里哺育着丁香……（打 4/5 分——有文采）
    C：春天是四季之一，位于冬季和夏季之间……（打 1/5 分——像百科全书）

  你知道了 B 风格受欢迎、C 风格不讨喜，下次写就朝 B 的方向靠。
  RLHF 就是这个过程的自动化+规模化版本。

▶ RLHF 的完整流程（只用三步记住）

  第一步：收集偏好数据
    给同一个 prompt，让模型生成多个回答，人类标注员排序：
    Prompt: "解释什么是黑洞"
    → A > B > C > D（A 最好，D 最差）
    大规模做这件事（数万条），得到一个"人类偏好数据集"。

  第二步：训练奖励模型（Reward Model）
    用偏好数据训练一个小模型——输入一个问题+回答，输出一个分数。
    这个分数模拟了"人类有多喜欢这个回答"。

  第三步：用奖励模型优化 LLM
    让 LLM 生成回答 → 奖励模型打分 → 朝得分高的方向调整模型参数 → 循环。
    这个循环叫 PPO（Proximal Policy Optimization），你不用管技术细节。

▶ RLHF 之后得到一个 **Chat Model（对话模型）**

  就是你现在用的 ChatGPT / Claude——会拒绝危险请求、
  回答简洁有用、能承认自己不知道。

五、这对你有什么用？（直接影响日常开发的 4 件事）
================================================

▶ 1. 理解了"Prompt Engineering 能解决什么、不能解决什么"

  Prompt Engineering 改变的是"模型的行为"，不是"模型的知识"。
  它在模型的第二阶段（SFT）和第三阶段（RLHF）已经确定的"性格"范围内做调整。

  能用 Prompt 解决的：
  ✅ 调整输出格式、语气、详略程度
  ✅ 加约束条件（"不要用专业术语"）
  ✅ 给 Few-Shot 示例 → 引导输出风格（Week 2 Day 2）

  不能用 Prompt 解决的：
  ❌ 让模型学会它在预训练时没见过的知识
  ❌ 修改模型底层的行为倾向（比如让一个保守模型变激进）
  ❌ 消除预训练数据中的偏见

  如果你有一个特定领域的私有知识库 → 用 RAG，不要想着 Fine-tune。
  如果模型在某个特定任务上格式/风格总是不对 → 考虑 Fine-tune。

▶ 2. 知道了"模型为什么有时候很怂"

  RLHF 阶段为了防止模型说错话，会让它对不确定的事"更保守"。
  但有时矫枉过正——你问一个完全无害的问题它也说"I apologize, but I cannot..."

  这种情况用 Prompt 解决："你是一个专业的事实性助手，请直接回答问题，不要过度谨慎。"

▶ 3. 理解了不同模型的"性格"差异从哪来

  GPT-4 vs Claude vs DeepSeek 的行为差异主要来自：
  - 预训练数据不同 → 知识覆盖面和语言偏好不同
  - SFT 数据质量不同 → 遵循指令的能力不同
  - RLHF 偏好标准不同 → "好回答"的定义不同（Claude 更保守，GPT-4 更直接）

  所以 Week 2 Day 5 你做的多模型评测是有意义的——不同模型对同一个 prompt
  反应不同，评测能帮你选最合适的模型。

▶ 4. Fine-tuning 不是银弹

  Fine-tuning 是在 SFT 基础上的继续训练。
  它适合：调整输出格式/风格、让模型学会特定领域术语。
  它不适合：注入全新的知识（Fine-tuning 后模型可能"背"下来但不能灵活运用）。

  如果面试官问："什么情况下用 RAG，什么情况下用 Fine-tuning？"
  你的答案：**知识更新用 RAG，格式/风格定制用 Fine-tuning。**


六、今天最重要的 4 个认知
==========================

1. 一个"能和人类对话的 LLM"不是一步训练出来的。
  它经历了三代教养：Pretrain（学语言）→ SFT（学规矩）→ RLHF（学品味）。

2. 幻觉不是 bug，是预训练的"天性"。
  预训练只学过"高频续写"，从没学过"我不知道"。
  RLHF 在尝试补救，但没有根治。

3. Prompt Engineering 操作的是"模型已学会的行为空间"。
  你在 SFT+RLHF 的既定框架内调整，不能超出模型的"训练记忆"。

4. RAG vs Fine-tuning 的边界很清楚：
  新知识/实时信息 → RAG；格式/风格/术语调整 → Fine-tuning。
  面试官问这个，答对了就加分。

八、速通模块总结：你学到了什么
==============================

这三天你从 LLM 的"黑箱"里看到了三个层面的东西：

  Day 1 — Tokenization：LLM 怎么"读"文字
    → token 才是计费单位，中文比英文贵，max_tokens 截断影响输出

  Day 2 — Transformer & Attention：LLM 怎么"理解"上下文
    → Self-Attention 让每个词关注所有其他词，O(n²) 成本是 RAG 存在的根本原因

  Day 3 — 训练流水线：模型怎么从"猜词"变成"对话助手"
    → 三代训练：学语言 → 学规矩 → 学品味，每个阶段留下不可消除的痕迹

有了这些，你不再把 LLM 当成"魔法盒子"来用——你知道它的能力边界从哪来、
知道遇到问题该用 Prompt 调还是用 RAG 补、面试问到"Transformer 是什么"
你能讲清楚 Attention 的原理。
