Week 2 Day 2 — Prompt 模板系统（Jinja2）学习日志
======================================================
日期：2026-06-11
主题：用 Jinja2 实现可变 prompt 模板，支持 few-shot 示例动态注入


一、为什么要用模板？
====================

痛点：写死的 prompt 字符串在代码里：
  1. prompt 和代码混在一起——改一个词要翻代码
  2. 同样的结构重复写——五要素每次都抄一遍
  3. Few-Shot 示例没法管理——塞 5 个例子变成字符串地狱
  4. 没法复用——翻译 prompt 的约束，总结 prompt 想用就得复制粘贴

模板引擎 = 把 prompt 当成"邮件合并"——
  你写好底稿，变量是填空位，数据填进去生成最终 prompt。


二、Jinja2 三个核心语法
========================

| 语法               | 含义           | 类比                      |
|--------------------|----------------|--------------------------|
| {{ variable }}     | 变量占位符      | Word 邮件合并的 <<姓名>>   |
| {% if %}...{% endif %} | 条件判断    | "VIP 加尊称，普通人加你好"   |
| {% for %}...{% endfor %} | 循环展开 | "给以下 5 人每人一封信"     |
| {{ loop.index }}   | 循环内计数器    | 生成"第 1 条、第 2 条"     |

区别：{% %} 是控制结构（if/for），{{ }} 是输出值。


三、今天建的 Prompt 模板系统
============================

文件结构：
  src/study_agent/prompt/
  ├── __init__.py          # 导出 PromptManager, FewShotExample, FewShotManager
  ├── templates.py         # PromptManager：加载 .j2 文件、渲染、管理
  ├── examples.py          # FewShotManager：存储示例、按标签筛选、格式化
  ├── demo.py              # 完整演示脚本（5 个场景）
  └── templates/
      ├── system_base.j2   # 基础模板：五要素骨架（可被其他模板复用）
      └── code_review.j2   # 代码审查模板：6 个变量 + Few-Shot 注入

核心类的职责：

  PromptManager
    - 从 templates/ 目录加载所有 .j2 模板
    - render(template_name, **variables) 一键渲染
    - FileSystemLoader 告诉 Jinja2 去哪找文件
    - Environment 是 Jinja2 的引擎核心

  FewShotManager
    - 存储 FewShotExample（input + output + tags）
    - filter_by_tags(tags) 按标签交集筛选
    - pick(tags, max_count) 筛选 + 限量
    - format(examples) 把示例列表转成 prompt 里能用的纯文本


四、code_review.j2 的变量设计
==============================

| 变量             | 类型  | 作用               | 对应五要素    |
|------------------|-------|--------------------|--------------|
| {{ role }}       | str   | 角色设定            | Role         |
| {{ context }}    | str   | 背景信息            | Context      |
| {{ rules }}      | list  | 审查规则（for 展开） | Constraints  |
| {{ output_format }} | str | 输出格式要求        | Format       |
| {{ few_shot_text }} | str | 预处理好的示例文本   | Few-Shot     |
| {{ code }}       | str   | 待审查代码          | Task         |

模板变量设计原则：
  - 字符串变量：直接 {{ }} 占位
  - 列表变量：+ {% for %} 循环展开，支持任意长度
  - 可选变量：+ {% if %} 条件，不传就不输出那段


五、Few-Shot 管理的关键设计决策
===============================

1. 为什么把示例和模板分离？
   示例是数据，模板是结构——分离后同一批示例可以被多个模板复用。

2. 为什么用标签筛选？
   50+ 个示例不能全塞进 prompt（token 爆炸）。
   按标签筛选 + 限制 3 个 = 用最少的 token 获得最大的示范效果。

3. 为什么注入前预先 format？
   模板只负责"在哪放示例文本"，不负责"怎么格式化示例"。
   format() 是 FewShotManager 的职责——单一职责原则。


六、今日核心收获
================

1. Jinja2 把 prompt 从"Python 字符串拼接"变成"模板文件 + 变量渲染"
2. 模板变量设计要对应 Day 1 的五要素：角色、上下文、任务、约束、输出格式
3. Few-Shot 示例注入是模板系统的核心价值——示例独立管理、按标签筛选、限制数量
4. 同一个 .j2 模板 + 不同变量 = 完全不同的 prompt（场景 4 演示了）
5. 模板引擎的核心：FileSystemLoader（眼睛） + Environment（大脑） + Template（手）
