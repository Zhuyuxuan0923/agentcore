二、每日回顾
============

Day 1 — 环境搭建
-----------------
  学了什么：Poetry 管理依赖、Black 格式化、Ruff 检查代码风格、
            MyPy 类型检查、Pre-commit 自动门卫
  踩过的坑：
    - pip install 慢 → 配了清华镜像源
    - 第一次知道"环境变量"是什么（.env 文件 + os.getenv()）
    - pre-commit 拦截提交 → 理解了"门卫模式"
  关键认知：工具帮你省时间，但你要先理解它在检查什么


Day 2 — FastAPI 速通
---------------------
  学了什么：FastAPI 路由、路径参数、Pydantic 类型校验、反例对比
  踩过的坑：
    - 请求 /bad-book/abc 时觉得应该返回"无效输入"，结果直接 500 崩溃
      → 理解了"没做参数校验"的后果
    - 路径参数没写类型注解（book_id vs book_id: int）→ FastAPI 不校验
  关键认知：FastAPI 的类型系统是"自动文档 + 自动校验"，不是装饰


Day 3 — OpenAI SDK 深度使用
----------------------------
  学了什么：messages 格式、system prompt 作用、temperature 调参、
            流式输出（streaming）、多轮对话、token 计数
  踩过的坑：
    - 把 system prompt 和 user message 搞混 → 模型不听使唤
    - temperature=0 和 temperature=1.5 输出差异巨大
      → 理解"确定性"和"创造性"的权衡
    - 流式输出用 requests 库卡住 → 知道后面要用 httpx（异步）
  关键认知：
    - system prompt = 给 AI 定规矩（角色、约束、输出格式）
    - temperature 不是"准确度"，是"随机度"


Day 4 — Anthropic SDK + 统一 LLMClient
---------------------------------------
  学了什么：Anthropic SDK 与 OpenAI SDK 的差异、封装统一客户端、
            支持 5 家 Provider 一键切换
  踩过的坑：
    - Anthropic 的 system 是独立参数（不是 message 数组的一项）
    - OpenAI SDK 能调 DeepSeek/GLM-4/Moonshot 因为它们兼容 OpenAI 格式
    - 写 if-else 判断 provider → 换成配置字典 4 行解决问题
  关键认知：
    - 抽象层 = 把差异关在笼子里，对外暴露统一接口
    - 加新厂商 = 在 PROVIDER_CONFIGS 加 4 行，不改业务代码


Day 5 — 异常处理与重试机制
---------------------------
  学了什么：try/except 基础、tenacity 库、指数退避、Rate Limit 处理、
            流式中断恢复、智能判断"什么错误该重试"
  踩过的坑：
    - 所有错误都重试 → Key 错了也重试 5 次（纯浪费）
    - 固定等 2 秒重试 → 服务器还在过载，你又打它一下
    - 异常捕获顺序错了 → AuthenticationError 被父类 APIError 吞掉
  关键认知：
    - 只重试"临时性"错误（网络抖动、限流、服务器故障）
    - 不重试"永久性"错误（Key 错了、参数格式不对）
    - 指数退避的本质：给服务器喘息时间


Day 6 — 项目骨架搭建
---------------------
  学了什么：Python 包结构、__init__.py 导出、@dataclass、TypedDict、
            @abstractmethod、Git/GitHub 基础操作
  踩过的坑：
    - 代码全堆在一起 → 按职责拆成 config/llm/tools 三个模块
    - __init__.py 里不写 import → 从外面导入要写很长路径
    - Git 的 user.name 没设 → commit 显示成 Claude
    - settings.local.json 包含 API Key 差点提交 → 补了 .gitignore
    - Windows SSL 证书问题导致 push 失败 → 切到 schannel
  关键认知：
    - 包结构 = 给代码加"隔层"，改一个模块不影响另一个
    - .gitignore 是"提交黑名单"，必须在第一次 add 之前配好

===================

  已掌握的工具/库：

    Black       — 代码自动格式化（line-length=100）
    Ruff        — 代码风格检查（E/F/I/N/W/UP 规则）

    - SDK 抽象层设计（配置字典 + 统一接口）
    - 异常层次结构（AuthenticationError ⊂ APIStatusError ⊂ APIError）
    - 指数退避（1s → 2s → 4s → 8s...）
    - Python 包的组织方式（__init__.py + 子模块）


四、踩坑 TOP 10（写文章素材）
=============================

以下是在本周学习过程中遇到的最有价值的坑，每个都可以写进文章。

1. API Key 硬编码在代码里
   现象：把 sk-xxx 直接写在 .py 文件里，git push 后全世界都能看到
   教训：用 .env 文件 + .gitignore，Key 永远不进仓库
   延伸：settings.local.json 也要 gitignore

2. 不区分"临时错误"和"永久错误"就盲目重试
   现象：Key 错了也重试 5 次，每次都等指数递增的等待时间
   教训：AuthenticationError(401) 和 BadRequestError(400) 重试没用，
        只有网络错误和 RateLimit(429) 才该重试

3. 固定等待时间重试导致"惊群效应"
   现象：所有请求同时失败 → 都等 2 秒 → 同时重试 → 服务器又炸
   教训：用指数退避 + jitter（随机偏移）分散重试时间

4. Anthropic 和 OpenAI 的 system 参数位置不同
   现象：把 system 放在 messages 数组里给 Claude → 不生效
   教训：封装抽象层时，要在初始化阶段消化掉 SDK 差异

5. 流式输出中断后从零开始而不是续传
   现象：收到一半网络断了，重连后又从头生成 → token 浪费
   教训：累积已收到的内容，重连时告诉 LLM "从这里继续"

6. Python 包里的 __init__.py 忘了写导入
   现象：from study_agent import LLMClient 报错找不到
   教训：__init__.py 是"对外的窗口"，要在里面做一层 export

7. Git 身份没配置导致 contributor 不是自己
   现象：git config user.name 为空，commit 显示其他人名字
   教训：装完 Git 第一件事：git config --global user.name + user.email

8. Windows 下 SSL 证书问题导致无法 push
   现象：Git 用 OpenSSL 找不到 Windows 证书库
   解决：git config --global http.sslbackend schannel

9. 类型注解没写导致 FastAPI 不校验
   现象：get_book(book_id) 和 get_book(book_id: int) 行为完全不同
   教训：FastAPI 靠类型注解做自动校验和文档生成

10. Pre-commit hooks 修改了代码但没重新 add
    现象：Hook（如 Black）自动改了格式，但你直接 commit → 格式没更新
    教训：Hook 修改文件后要重新 git add，或者等 hook 通过后再 commit
